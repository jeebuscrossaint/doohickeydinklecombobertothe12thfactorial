Overview

Characterizing a photonic lantern requires acquiring interference pattern images across multiple output ports and optical wavelengths, then reconstructing the electric field from each hologram using Fourier optics. Done manually, a single session takes several hours of careful alignment and tuning.

This project automates the entire pipeline — hardware control, fringe optimization, data collection, and field reconstruction — reducing operator time to minutes. Built for Dr. Eikenberry's group at UCF CREOL, based on prior testbed work by Caleb Dobias and Miguel Romer.

Hardware

  • HP 8168E Tunable Laser — 1475–1575 nm, GPIB control
  • Dicon GP700 Fiber Switch — selects photonic lantern output port, RS-232
  • Thorlabs Motorized Polarization Controller — 3-axis stepper motors, USB
  • Xenics Bobcat 320 GigE Camera — InGaAs sensor, captures hologram frames

Features

  • Single-command connection and safe shutdown of all four hardware devices
  • Automated (legs × wavelengths) acquisition loop — 49 images in ~30 minutes unattended
  • Real-time fringe visibility detection (variance, Michelson contrast, or FFT peak methods)
  • Automatic polarization optimization via motor grid search when fringes are poor
  • FFT-based holography: twin image extraction, IFFT field recovery, optional defocus correction
  • LP mode decomposition across 7-port lantern with fidelity metrics
  • 12-panel analysis plots per hologram (raw → FFT → field → modes → power distribution)
  • GUI (Tkinter) and CLI interfaces; long-running operations run in background threads
  • Single YAML config for all hardware addresses, experiment parameters, and processing options

How it works

The collection loop iterates over each lantern port and each wavelength. For every combination it sets the laser, captures a frame, and checks fringe visibility. If the fringes are below threshold, the polarization motors perform a grid search to find the best alignment before saving the image and metadata.

Processing runs the saved holograms through a 2D FFT, isolates one sideband in Fourier space, and inverse-transforms to recover the complex electric field. That field is then projected onto an LP mode basis to produce the mode decomposition coefficients and plots.

Stack

  • Python 3.8+ — core implementation
  • NumPy / SciPy — FFT, signal processing, mode decomposition
  • PyVISA + GPIB-ctypes — laser instrument control
  • PySerial — RS-232 fiber switch control
  • ctypes — Thorlabs Kinesis DLL and Xenics SDK bindings
  • Matplotlib — visualization and analysis plots
  • PyYAML — configuration management
  • Tkinter — GUI interface
